# m-page-archive

## 项目简介

`m-page-archive` 是一个用于归档和发布多个独立 HTML 页面的通用仓库。  
本仓库不针对任何特定域名或网站，仅作为静态页面文件的集中存储与展示空间，便于长期保存与分享。

## 目录说明

```
m-page-archive/
├── pages/          # 存放所有归档的 HTML 页面文件
├── assets/         # 页面依赖的静态资源（图片、样式、脚本等）
├── index.html      # 可选的总览页面，用于索引所有归档页面
└── README.md       # 本文件
```

- `pages/` 目录下每个 HTML 文件为一个独立归档页面，文件名建议使用有意义的标识符。
- `assets/` 目录按需组织，可包含子目录以区分不同页面的资源。

## 页面归档说明

每个归档页面均为完整可用的 HTML 文档，包含独立的样式与脚本。  
归档内容可能包括但不限于：信息页面、工具页面、演示页面、存档页面等。  
所有页面均不涉及商业推广、用户数据收集或诱导性操作。

### 添加新页面

1. 将 HTML 文件放入 `pages/` 目录。
2. 如有依赖资源（图片、CSS、JS 等），放入 `assets/` 对应子目录。
3. 可选：更新 `index.html` 或本 README 中的索引列表。

### 页面要求

- 文件编码建议使用 UTF-8。
- 页面内部链接建议使用相对路径，以保持仓库内可移植性。
- 避免引用外部不可控资源（如外链字体、脚本），确保离线可用。
